secV
Documentation

secV is a modular offensive security framework with a REPL shell, tab-completion, and Linux-native navigation. Each module is a self-contained tool loaded on demand.

Install git clone https://github.com/secvulnhub/secV && cd secV && bash install.sh

Shell Reference

secV Shell

Module Commands v2.3+
Loading Modules
use <module>Load a module by name (e.g. use netrecon)
backUnload the current module and clear params. Equivalent to cd ..
reloadRescan the tools directory for new or updated modules
Configuration
set <param> <value>Set a module parameter. First token after set is the key. Example: set operation backdoor_apk
unset <param>Clear a parameter
show optionsList all parameters with types, required flags, and current values
Execution
run <target>Execute the loaded module against target (IP, hostname, device, etc.)
Information
show modulesList all available modules grouped by category
info [module]Module details, version, author, and dependency status
search <keyword>Search modules by name, description, or category
help moduleShow full help for the currently loaded module
sessions [list|interact <id>|kill <id>]Manage active Meterpreter / shell sessions
System
updatePull the latest version from git (runs update.py)
clearClear the terminal screen
exit / quitExit secV
Linux Navigation native passthrough

All standard Linux commands work natively inside secV. The shell tracks a working directory so commands like ls, cat, and find operate on the correct path. Use cd .. to navigate up — if a module is loaded, this unloads it (same as back).

Directory Navigation
cd <dir>Change working directory
cd .. / cd ../Go up one level. If a module is loaded, acts as back
cd ~Change to home directory
pwdPrint current working directory

Prompt Reference

The prompt adapts to show what is loaded and how many parameters are currently set. 

# No module loaded
secV ❯

# Module loaded, no params
secV android_pentest ❯

# Module loaded, operation set
secV android_pentest › backdoor_apk [2 params] ❯

# Module loaded, custom operation
secV netrecon › full [3 params] ❯

Getting Started

Quickstart

01
Install secV
Clone the repo and run the install script. It detects your distro and offers to install missing dependencies automatically.
git clone https://github.com/secvulnhub/secV
cd secV
bash install.sh
02
Launch the shell
Run the secV binary. It auto-scans the tools directory and loads all modules.
./secV        # from the repo directory
secV          # if installed system-wide
03
Load a module
Use use <name> to load any module. Tab completes module names.
use netrecon
show options
04
Configure and run
Set parameters with set <param> <value>. The first token after set is always the parameter name — not the module name.
# Network scan example
set mode full
set scanner nmap
run 192.168.1.0/24

# Android backdoor example
use android_pentest
set operation backdoor_apk
set package com.instagram.android
run connected
05
Navigate like Linux
Use back or cd .. to unload a module. Regular Linux commands like ls, cat, grep, mkdir work natively.
ls tools/network/
cat tools/network/netrecon/module.json
back            # or: cd ..
